// the_problem

flags shouldn't outlive
your sprints

[x] no_tests

flags ship without any test coverage. you find out they're broken in production.

[x] vendor_lock-in

your flags live in a saas dashboard. change plans and lose your entire config history.

[x] stale_flags

flags pile up. nobody knows which are safe to delete. the codebase rots.

// how_it_works

three commands. total control.

01. write your flagfile

define flags using the clean dsl. segments, environments, return values — all in one file next to your code.

02. $ ff check

validate syntax, run 22 lint rules, and execute inline @test annotations — all in one command.

03. evaluate anywhere

use the rust crate, typescript library, or http api. identical behavior across all runtimes.

// more_capabilities

built for the full lifecycle

> percentage() gradual rollouts

bucket users by stable hash. roll out to 10%, then 50%, then 100% — no code changes required.

> ff_serve http api server

built-in http server with openfeature compatibility. evaluate flags from any language over the network.

> json_returns rich return values

flags don't have to return booleans. return json configs, strings, or numbers to drive full feature configurations.

> ff_find codebase search

find every usage of a flag across your codebase. detect unused flags before they become dead weight.

> push / pull distributed sync

sync flagfiles across services and environments. push updates centrally, pull remotely. offline-capable.

> wasm browser + edge

compiled to wasm. evaluate flags at the edge, in the browser, or in any wasm runtime — with identical semantics.
